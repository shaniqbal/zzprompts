## SECTION 1: Main File Organization & Code Quality

### 1.1 File Structure & Organization
- [ ] All files organized in logical directory structure
- [ ] Common elements grouped and clearly labeled
- [ ] File naming follows consistent convention (lowercase, hyphens)
- [ ] No development files included (.git, .sass, .scss source, node_modules, .DS_Store, Thumbs.db)
- [ ] No backup files included (~, .bak, .tmp)
- [ ] All paths use relative references (no absolute paths)
- [ ] Maximum editability achieved for end users
- [ ] Code follows latest industry best practices
- [ ] Code fully validated (no syntax errors)

### 1.2 WordPress Specific Files
- [ ] `style.css` header complete with all required fields
- [ ] Theme name in style.css matches submission name
- [ ] Version number consistent across all files
- [ ] Text domain matches theme slug exactly (`zz-themename`)
- [ ] `functions.php` properly structured
- [ ] Template files follow WordPress hierarchy
- [ ] No hardcoded database table prefixes

### 1.3 Code Comments & Readability
- [ ] Critical functions documented with inline comments
- [ ] Complex logic explained for future developers
- [ ] File headers include purpose/description
- [ ] TODOs and FIXMEs removed
- [ ] Commented-out code blocks removed
- [ ] Code properly indented (consistent tabs/spaces)

## SECTION 2: Asset Licensing & Attribution

### 2.1 License Documentation Directory
- [ ] `/Licensing/` directory created in root of ZIP
- [ ] GPL license .txt files included (for Joomla, Drupal, OpenCart, osCommerce)
- [ ] OSL v.3 license .txt files included (for Magento, PrestaShop)
- [ ] WordPress themes: License auto-attached (no manual licensing needed)
- [ ] All license files properly named and formatted

### 2.2 Image Assets
- [ ] All demo images have commercial redistribution licenses
- [ ] All preview-only images have commercial licenses
- [ ] Image sources documented in `/Licensing/image-sources.txt`
- [ ] Attribution requirements listed for each image
- [ ] Preview-only images clearly marked in item description
- [ ] No copyrighted/trademarked images without permission
- [ ] No celebrity/recognizable people without model releases

### 2.3 Font Licenses
- [ ] All fonts have commercial redistribution licenses
- [ ] Google Fonts used (free, no license needed) OR
- [ ] Premium fonts licensed and documented
- [ ] Font sources listed in `/Licensing/font-sources.txt`
- [ ] Attribution requirements noted
- [ ] Webfont formats included (woff, woff2)

### 2.4 Icons & Graphics
- [ ] Icon libraries have commercial licenses
- [ ] Custom icons created or licensed
- [ ] SVG files optimized and cleaned
- [ ] Icon sources documented
- [ ] Attribution requirements noted

### 2.5 Third-Party Code & Libraries
- [ ] All JavaScript libraries have GPL-compatible licenses
- [ ] All PHP libraries/frameworks properly licensed
- [ ] Library versions documented
- [ ] Sources and licenses listed in `/Licensing/code-sources.txt`
- [ ] No obfuscated/minified-only libraries (source must be available)

### 2.6 Demo Import Content
- [ ] One-click demo import content licensed for redistribution
- [ ] Preview-only content clearly marked in description
- [ ] Placeholder images used where redistribution not allowed
- [ ] Demo import documentation includes asset disclaimers

### 2.7 Security & Malware Check
- [ ] Full malware scan completed (Sucuri, VirusTotal, etc.)
- [ ] No malicious code detected
- [ ] No backdoors or unauthorized data transmission
- [ ] No encoded/obfuscated code (unless necessary and documented)
- [ ] All third-party code from trusted sources

## SECTION 4: Item Title & Branding

### 4.1 Title Structure (MANDATORY FORMAT)
- [ ] Brand name at beginning: `ZZ [Brand Name]`
- [ ] Separator used: `|` (pipe)
- [ ] Descriptive subtitle follows
- [ ] Format: `ZZ BrandName | Descriptive Subtitle`
- [ ] Example: `ZZ Vertex | Modern Business WordPress Theme`

### 4.2 Title Content Requirements
- [ ] WordPress themes include "WordPress Theme" in title
- [ ] HTML templates include "HTML Template" in title
- [ ] Email templates include "Email Template" in title
- [ ] Template kits include "Template Kit" in title
- [ ] Item type clearly identifiable from title

### 4.3 Site Builder Claims (Critical)
- [ ] "Site Builder" / "Website Builder" used ONLY if you created the builder
- [ ] Third-party builders (Elementor, WPBakery) = use "Includes Site Builder"
- [ ] No misleading builder claims in title
- [ ] Builder type specified if included

### 4.4 Title Restrictions - Prohibited Content
- [ ] NO keyword stuffing (multiple related keywords)
- [ ] NO repeated words
- [ ] NO subjective terms (best, amazing, brilliant, perfect, ultimate)
- [ ] NO misleading/impersonating names (Avada → Avaada)
- [ ] NO "Pro" in title (unless lite version on WordPress.org exists)
- [ ] NO non-English words (except common expressions: Carpe Diem, Bon Voyage)
- [ ] NO trademark violations
- [ ] NO competitor name references

### 4.5 Title Quality Checks
- [ ] Title is clear and descriptive
- [ ] Title is concise (not excessively long)
- [ ] Title is professional (no gimmicky language)
- [ ] Title accurately represents the theme
- [ ] Title is memorable and brandable
- [ ] Title is unique in ThemeForest marketplace

### 4.6 Title Examples - Review These

**✅ ACCEPTABLE:**
- `ZZ Revirta | Virtual Assistant WordPress Theme`
- `ZZ Lami | Recipe Blogging WordPress Theme`
- `ZZ Nexus | Corporate Business HTML Template`
- `ZZ Pixel | Creative Portfolio WordPress Theme`

**❌ UNACCEPTABLE:**
- `Photography - Photo Theme for Photographers` (keyword stuffing)
- `ZZ Kapow | Karate, Ninja, Kickboxing, Sports HTML` (keyword stuffing)
- `ZZ Avaada | Responsive Theme` (impersonation)
- `ZZ Vertex | The Best WordPress Theme` (subjective terms)
- `ZZ ProBusiness | WordPress Theme` ("Pro" without lite version)

## SECTION 5: Final Package Assembly

### 5.1 ZIP File Structure
```
zz-themename.zip
├── zz-themename/               (Main theme folder)
│   ├── style.css
│   ├── functions.php
│   ├── index.php
│   ├── [all theme files]
│   └── ...
├── Licensing/                  (License documentation)
│   ├── license.txt
│   ├── image-sources.txt
│   ├── font-sources.txt
│   └── code-sources.txt
├── documentation.pdf           (Help file)
└── [optional-child-theme.zip] (If child theme provided)
```

### 5.2 Package Contents Checklist
- [ ] Main theme folder named correctly: `zz-themename`
- [ ] `/Licensing/` directory at root level (not inside theme folder)
- [ ] `documentation.pdf` or `documentation.html` at root level
- [ ] Demo import files included (if applicable)
- [ ] Child theme included (optional but recommended)
- [ ] README.txt with quick start guide (optional)

### 5.3 File Size Optimization
- [ ] Total ZIP size under 2GB (recommended limit)
- [ ] Images optimized for web (use TinyPNG, ImageOptim)
- [ ] Unnecessary files removed
- [ ] Large assets documented (if unavoidable size)
- [ ] No duplicate files/libraries

### 5.4 Preview Images Preparation
- [ ] Live preview images watermarked (`© ZZ Themes` or similar)
- [ ] Watermark subtle but visible
- [ ] Watermark doesn't ruin user experience
- [ ] High-quality screenshots prepared
- [ ] Mobile/responsive views shown

### 5.5 Item-Specific Requirements

**For Email Templates:**
- [ ] Tables used for layout (not divs)
- [ ] All CSS inlined
- [ ] Tested across email clients (Gmail, Outlook, Apple Mail, etc.)
- [ ] CampaignMonitor compatibility verified
- [ ] MailChimp compatibility verified
- [ ] Responsive design tested on mobile email apps

**For PSD Templates:**
- [ ] All layers named clearly
- [ ] Layers organized in logical groups
- [ ] Smart objects used where appropriate
- [ ] Text layers editable (not rasterized)
- [ ] Effects preserved (not flattened)
- [ ] Color swatches provided
